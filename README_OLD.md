#### User (Пользователи)
- `id` - первичный ключ
- `login` - логин уникальный, максимум 12 символов
- `password` - первичный ключ

- `is_active` - при создании True, Альтернатива удаления
- `fullname` - полное имя пользователя, максимум 150 символов

## Функционал кода
### Работа с пользователями
отвечает UserController
```python
# регистрация пользователя администратор
from Controllers.UserController import UserController
UserController.registration(
        login='user',
        password='user'
    )
# вывод списка пользователей
for row in UserController.get():
    print(row.id, row.login, row.password, row.role, row.is_active, row.fullname)
# обновить данные пользователей
UserController.update(2,login = "admin2")
```

### Работа с категориями
отвечает CategoryController
```python
from Controllers.CategoryController import CategoryController
# Создание категории
CategoryController.create("Проблемы с сетью")
# Получение списка категорий
for category in CategoryController.get():
    print(category.id, category.name)
# Обновление категории
CategoryController.update(1, name="Новое название")
# Удаление категории
CategoryController.delete(1)
```

### Работа с заявками
отвечает TaskController
```python
from Controllers.TaskController import TaskController
# Создание заявки
TaskController.create(
    topic="Проблема с доступом",
    description="Не могу зайти в систему",
    user_id=1,
    category_id=1,
    priority="Высокий"
)
# Получение заявок пользователя
for task in TaskController.get_by_user(1):
    print(task.id, task.topic, task.status)
# Изменение статуса заявки
TaskController.change_status(1, "В работе")
# Фильтрация заявок
for task in TaskController.filter_tasks(status="В работе"):
    print(task.id, task.topic)
# Статистика
stats = TaskController.get_statistics()
print(f"Всего заявок: {stats['total']}")
```

## Графический интерфейс пользователя

### Управление пользователями (Tkinter)

Система теперь включает графический интерфейс для управления пользователями, реализованный с использованием Tkinter.

#### Структура графического интерфейса:

```
Views/
├── UserView.py              # Главное окно управления пользователями
├── AddUserWindow.py         # Окно добавления нового пользователя
└── EditUserWindow.py        # Окно редактирования существующего пользователя
```

#### Основные функции интерфейса:

1. **Главное окно (UserView.py)**:
   - Отображение списка пользователей в таблице Treeview
   - Кнопки для добавления, редактирования, удаления пользователей
   - Изменение статуса активности пользователей
   - Автоматическое обновление таблицы при изменениях
   - Выбор пользователя щелчком по строке таблицы

2. **Окно добавления пользователя**:
   - Ввод логина (максимум 12 символов, уникальный)
   - Ввод пароля с подтверждением
   - Выбор роли (Пользователь, Специалист, Администратор)
   - Ввод полного имени (опционально)

3. **Окно редактирования пользователя**:
   - Редактирование логина, роли и полного имени
   - Валидация вводимых данных
   - Сохранение изменений в базе данных
